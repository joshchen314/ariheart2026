title: 以巨集變數(Macro)處理普查問卷，討論一個例子
---

巨集可說是讓人望而生畏，原因在於她根本跟 DATA 階段與 PROC 階段不太一樣，風格迥異，讓SAS的使用者相當疑惑，至少對我來說。我想，可能的主因在於，Macro 的宣告。因為，只要以正確的方式宣告 Macro 後，就可以一直調用之。

現在就來先討論第一個案例，先假設有一個表格：

|  |  |  |
| --- | --- | --- |
| D062041 | D063041 | c |
| 1500 | 15 | 2 |
| 2000 | 14 | 1 |
| 1010 | 21 | 1 |

這是針對一位調查對象，一個問卷中的問題，蒐集相關的資料，之後，要檢測填表的狀況，如是否有漏填、是否有填寫不完整等等，許多的檢誤條件可以設定。這裡的例子先檢測**是否完整填寫**。

我們可以發現，(1500,15,2)是完整填寫的例子，這是正確的，每一筆叫做 列觀測值。若要檢測這個條目的完整性，可以這樣做：\
IF D062041 \* D063041 \* c =0 THEN ......;\
當相乘等於零時，很明顯的有漏填的情況。(當然需要先令若遺漏時設為零) 並且若有遺漏，需要計算數量，這樣就可以知道有多少不完整了。

這個表格有 3 個條目，那就需要這樣的程式碼執行 3 次。若是 10, 30, 100次時，就會很傷腦筋了。這時候就可以使用 MACRO 了。如下：

|  |
| --- |
| **%MACRO** A3(F,A,B,C,D,E);  /\* 問項  不完整\*/ …. %DO i=**1** %TO &B/\*17\*/ … %if **1**<=&i<=**9** %then %do;  /\*   如果項目超過 10 ，有進位問題\*/ %DO J=&C %TO &D … IF &F.0&A.&J.0&I.1>**0** THEN KK&E.&i.&D=KK&E.&i.&D+1; ELSE KK&E.&i.&D=KK&E.&i.&D+0;KL&E.&i = KL&E.&i   +  &F.0&A.&J.0&I.1; IF KL&E.&i>0 THEN DO;IF **KK&E.&i.&D<&D** THEN …; … END; %end; %end; %IF **10**<=&I<=**99** %then %DO; …. %END; TUTU&e=(TUTU&e+CN&E.&i); %END;/\*17\*/ CNTZ&E=TUTU&e **%MEND** A3;   /\* 問項  不完整\*/ %***A3*(D,6,3,2,4,8);**/\*農地不完整\*/ |

這裡的想法是，如果列觀測值有任何一個欄位有填寫（有值，且非零），也就是每一個欄位相加大於零。 那麼，我非常希望每一個欄位都要填寫非零的值，也就是說全部欄位相乘不能等於零；很明顯的，只要有一項未填寫，就會使全部欄位相乘等於零。

但是，我遇到一個問題，暫時找不到一個方法可以在巨集裏面以 Loop 循環相乘。所以，我以加法來克服（或是逃避，sadly）問題：在欄位數固定的前提下（另為 n），只要欄位裏面有值就加 1 ，欄位沒有非零的值就加 0 。如果最後累加的總值等於 n ，則列觀察值完整填寫；如果最後累加的總值不等於 n ，則列觀察值並未完整填寫，有缺漏。

%***A3*(D,6,3,2,4,8);\**這裡可以分解一下。 總共有六個參數，第 1 個參數控制變數的第一個值，這裡的例子是普查資料庫裡的表號。第 2 個參數控制第幾問項。第 3 個參數控制有幾個列觀察值。第 4 個參數控制從哪一行開始，第 5 個參數控制哪一行結束。第 6 個參數控制，（加總錯誤）變數的名字。

因此，在大費周章的 宣告 巨集後（**%MACRO** A3(F,A,B,C,D,E); ），並且辛苦設定好那個參數控制那個東西後，之後只要遇到檢測 是否完整填寫表格，就可調用 A3巨集了。
