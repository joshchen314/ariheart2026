其實，應該是自己沒學好，加上懶惰，也不全然是Windows的錯。我會這麼想用版本控制不是因為有宏大遠光，而是因為要交 Coursera 的作業。所以，查了許多教學，發現怎麼都是高手在用版本控制（因為直接使用命令列），搞的我心萌退意，Coursera 的課也就放著了。突然間，我發現今年快要過了，可不想一事無成度過完2015年，只好乖乖來學版本控制交作業。難題是就一個重度依賴滑鼠的人要怎麼完成版本控制呢？
第一步，把應該安裝的程式裝好，命令介面版本控制：Git、Github；非命令介面：TortoiseGit。前二者，是由命令介面控制的，可以體會到版本控制的精妙之處。但是，我只是要交作業而已，所以後面的 TortoiseGit 就能滿足我的需求。即使如此，還是將前二者裝好，以免有朝一日可以使用。
第二步，版本控制交作業：以下紅字標明的**五件事**
1. 要先取得 Github 的資料夾位置，也就是Github網站指明的 HTTPS Clone URL。
2. 在本機桌面按右鍵後，選擇 「 Git Clone 」，對話框的URL就填上面的 HTTPS Clone URL，directiory 就選擇本機的目的資料夾。到這裡就完成了複製了，也就是說，建設好Github資料夾與本機資料夾的連結。
3. 本機資料夾裡，新增了寫好的作業。那要如何推送到Github資料夾呢？首先，在本機資料夾按右鍵，執行 「Git Commit ->"master"」 。第二，在本機資料夾按右鍵，選 「TortoiseGit」，再執行 「push」 ，接著就按照指示填入帳號密碼。這樣就完成了版本控制交作業。
在完成這樣的流程後，就比較有版本控制的概念了，也發現如果程式碼都是以這個方法儲存，我真的就不用在檔名加上版本次數或日期了，並且也會很清楚程式碼的更動過程。總之，希望下一步是將命令列學起來。
